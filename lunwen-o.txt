创慧公司内部管理系统的设计与实现

第 1 章 绪论
1.1 研究背景
为实现具体的管理目标和管理对象，完善组织的内部管理体系。子系统应遵循项目管理系统的总体目标。各子系统的作用不是横向的，各子系统在实现总体目标中起着主导作用。对这样的系统，可以横向进行不同分析，对不同行业的项目管理进行综合研究[1]。
随着科学技术的不断发展，在公司的传统管理模式中，上游管理模式需要纸质材料，需要多个管理层的签名和参与。其中，纸质材料的传输过程只需要花费大量的时间和人力资源。为了适应新时代的演进，提高公司的工作效率，减少工作时间和成本，便于更好地管理公司，将纸质材料转变为信息自动化管理已成为时代潮流。
公司管理系统是一套相对便利的软件，它根据公司各项业务的特点【2】，结合计算机信息化管理的优势，利用计算机网络以及其他软件工具对公司业务快速进行便利管理的一个系统软件。它可以将许多手工制作且方式繁琐的管理工作通过信息化管理改由计算机的高效形式进行，降低公司员工的劳动效率、劳动强度以及劳动压力，并且使公司的各项数据保存更加方便与安全，将公司变得更加信息化，使其日益壮大。本系统完整、全面、快速、高效地收集整理整个公司业务的有关信息，而且具有不同人员不同系统权限的管理方式。针对目前复杂的市场环境，这款系统能够契合公司的需求，并且具有广阔的市场前景和高效的获利能力。
1.2 国内外研究现状
1.2.1 国外研究现状
20 世纪 70 年代以来，国外的公司治理制度得到了发展，尤其是在英国、美国等发达国家，这些国家特别重视公司治理制度的发展和研究，在理论上取得了创新和突破。进入 21 世纪后，软件技术和控制的便捷性已逐渐成为公司管理系统的基本特点。其中，Twitter 的业务管理系统最为著名。该系统主要采用 Oracle 数据库，具有方便、简单、用途明确的特点，其优点已成为国外企业管理系统的参考模板之一[3]。
在互联网应用方面，大多数国家经常参考欧洲和美国的先进发展经验。原因是过去 30 年的 IT 革命在欧洲和美国的繁荣中起到了主导作用。欧洲和美国在信息技术和互联网技术方面非常先进，并将计算机应用于生活的各个领域。特别是公司管理系统，

在国外很多公司应用计算机技术几乎是很普遍的。每个企业的管理模式不一样，寻找一个适合发展方向的公司管理系统有助于公司经济效益的提高，促使企业获得竞争优势，实现公司竞争力的提高。
1.2.2 国内研究现状
在过去几年，我国互联网行业迎来井喷式爆发，许多行业受到信息技术的强烈冲击，在中国庞大的人口基数和迅速上升的互联网普及率，创造出世界范围内的“现象级市场”——中国互联网行业。这极大地促进了许多公司的信息化进程，使公司管理系统在国内的公司中出现热潮。
特别是国家大力支持企业采取多种途径建设和配套公司管理系统，在过去的基础上，本着高效便捷、上传下达、公平发展的原则，提速公司管理系统的发展。针对过去我国在公司管理系统领域的发展缓慢，参考国际标准，结合中国实际，使越来越多优秀的公司管理系统在国内广泛应用。许多公司针对自身情况陆续推出针对企业规模、企业类型不同的管理系统，其中以金蝶等相关产品最为大众熟知。

2.1 B/S 结构介绍
B/S 结构是为了让浏览器和服务器更好地满足人们的需求，使两个工具能够在以前繁琐的操作，让开发人员减少负担，降低企业的开发成本，为互联网程序的开发减轻了时间成本和资金压力 [7]。根据目前的市场情况，在技术论坛中我们可以看到，多名开发者在利用了这个结构，使他们的学习成本进一步降低。例如在数据库的开发上，我们可以看到，利用这种技术访问数据库比较容易掌握与操作，当不同的人想要操作时，可以操作。甚至在不同平台也可以操作，极大地增强了这个结构的广泛应用，促使其更好地发展。
2.2 MySQL 数据库介绍
MySQL 是一个轻量级数据管理系统。它具有良好的性能价格比，被称为“最优秀”的关系数据库系统之一，并且已经成为全球范围内使用最为广泛的关系数据库系统。同时,My SQL 也被公认为是最先进的关系数据库系统[8]。目前 My SQL 已广泛应用于中小型互联网网站。MySQL 它的性能十分强大，可以为用户保障系统的安全，支持存储和处理较大的数据，使数据能够高效运行。
2.3 Spring Boot 框架介绍
Spring Boot 为 Sprin 应用提供自动安装功能功能。通过使用该工具，就能轻松地将自己的软件部署到一个可以在不同平台通用的环境。了解一下如何设置所需的配置参数，尽快地开始应用程序开发的开发[9]。以下是 spring boot 的一些特性：如嵌入式 servlet 容器、Tomcat、Bridge 提供自动配置的“启动器”项目对象模型（POMS)、 spring 容器的自动装配、日常监控、web 容器、健康、安全等通用功能；以及完全不需要生成代码。

2.4 Spring Security 介绍
Spring Security 是在互联网上广泛使用的安全框架，对基于 spring 的程序提供安全解决方案。它提供了一系列 bean，可以在 spring 应用程序的上下文中进行配置，并使用 spring IOC（控制反转）、di（依赖注入）和 AOP（面向方面编程）功能。为了更好地使应用具有安全性，它为 Java 程序提供了良好的安全保障，使 Java 程序能够在多种不同的条件下得到完全可靠的安全保障。它为应用系统提供了声明式的声明式安全访问控制功能，减少了企业系统安全控制的重复代码编写[10] 。Spring Security 提供全面和可扩展的身份验证和授权支持、针对固定会话的保护、点击劫持、跨站点请求伪造和其他攻击、支持 Servlet API 集成以及支持与 Spring Web MVC的集成。

IDEA,全名 Intelli JIDEA,是 Java 开发的一个优秀平台。为了方便开发者对 Java程序的开发，技术人员专门开发了这款 Java 开发程序软件。在代码智能助理、代码错误提示、代码重构、J2EE 支持、支持 Git 工具、支持 Svn 工具、JUnit、CVS 集成、代码分析、创新 GUI 设计等方面[11]。在其中最突出的功能是智能调试，它调试 Java代码、Java Script、JQuery、Ajax 和其他技术实现了一个突破。例如，如果查看 Map类型的对象，而实现类使用哈希映射，则自动筛选空条目实例。其次，需要表达式的值来动态求值，例如，如果我得到一个类的实例但不知道它的 API，我可以对它代码完成以指示它支持的方法。最后，在多线程调试中，需要检查多线程执行也可以使用相关工具。

Maven 是互联网上的一个开源项目，它是一个创新的项目管理工具。在之前的程序开发中，经常出现各种问题。它用于 Java 项目构建、依赖关系管理和项目信息管理。Maven 包括一个项目对象模型、一组标准、一个项目生命周期、一个依赖关系管理系统以及实现生命周期阶段定义的插件目标的逻辑。项目 Maven 需要配置仓库[13]。对于项目 a、项目 b 等通过 Maven 从远程仓库（网络中的仓库）下载 jar 包并将它们保存到本地仓库。本地仓库就是本地文件夹。如果第二次需要这个 jar 包，本地仓库就以及备份。本地存储可以解释为缓存。对于本地仓库，您不必每次都从远程仓库下载。Maven 有三个独立项目生命周期。需要注意的是，这“三句话”是“三句分开”的。这三组生命周期是：清洁生命周期，以便在实际施工前进行一些清理工作；标准生命周期设计的核心组成部分，包括汇编、测试、包装和部署；以及生成项目报告网站和生命周期。

3.3.5 财务管理

3.3.6 后勤管理
后勤管理人员进行公司活动的安排和保修的资料重要参考是财务管理，这同样是系统的重要功能之一。后勤管理系统主要是后勤管理人员对公司活动和保修安排等后勤信息进行记录，让后勤管理人员对公司的后勤情况进行充分了解，同时，后勤管理人员还负责公司的日常投诉处理和信箱处理。
3.3.7 销售管理
销售管理人员进行公司产品查看与宣传的计划的重要参考是销售管理，这对系统十分重要。该模块主要是销售管理人员对产品的详细信息和宣传的安排等销售信息进

行记录，让销售管理人员对公司的销售详细情况进行充分了解。
3.4 非功能性需求分析
系统安全包括反馈数据库、对数据库的数据进行安全保存，保证其能够完整地储存，为防止非系统用户更改系统里的信息，让不明身份的人对系统产生重大威胁，从而确保系统安全。
这种身份验证通过为用户提供统一的身份识别方式，然后将来自合法用户的数据存储在系统中。当一个合法的用户登录时，系统会自动验证信息，最后用户可以在系统运行前验证信息，公司的管理系统会介绍每个用户是如何通过上述方法进入系统的。若未经过验证，则系统会向用户发送提示信息；若验证成功，则系统将会直接进入正常工作状态；若未经验证，则系统无法启动，此时会自动停止工作。如果输入不通过，将提示该消息，用户必须再次输入该消息，并且输入三次以上，系统自动退出。
为了使系统更加实用，用户界面以友好的方式设置，并且系统的操作相对简单。整个界面简洁明快，可以有效减少用户的错误。用户认为，如果他们想修改数据或将数据输入系统，系统将相应地提示。假如需要对系统的数据进行查看，就会生成相应页面，从而方便了用户的使用。通过这些方法使得该系统更加完善和实用，使用户能够快速适应新的系统。

第 4 章 系统详细设计
4.1 系统的设计目标与原则
4.1.1 系统的设计目标
对用户的需求进行具体分析之后，系统将有六个功能，我将公司管理系统分别分成了行政管理、人力管理、财务管理、后勤管理、销售管理、系统管理这六个互不干扰的模块。
行政管理简单来说是一个上传下达的关键功能，它能有效地将上级的要求通知给其他部门，同步到整个系统，使各部门能够更好地清楚公司的服务需求和营销需求，让行政人员简单快捷地使用。
人力管理是特殊且重要的，公司管理系统利用这些数据可以快速地查看员工信息、假期时间、工资信息等信息，了解员工在公司的各项情况，更好地提升员工在公司的获得感。
公司通过财务管理这个功能管理公司的财务信息，通过资金的收入与支出，分析公司在当前经营环境下的盈利与亏损情况，查询相关信息，制定财务方案，使公司更加针对性的进行资金的收支[18]。
一般情况，后勤管理人员负责后勤管理这个模块。一旦出现相关情况，后勤管理这个功能模块将自动更新相关信息，包括保修列表、投诉信息与信箱信息。了解活动信息能根据活动安排快速地制定相关计划，了解保修信息可以快速地使公司物品得到相应的维护，了解投诉信息可以更加地反应员工受到的不正当对待，了解信箱信息可以快速地接受到员工对公司的各种信息，更好地维护公司这个平台。
销售人员通过销售管理这个功能模块，可以快速地录入产品信息与宣传信息，通过对产品信息的分析，快速地制定产品的宣传计划，帮助公司更好地卖出更多产品，取得更大的销售额。
系统管理这个功能将被管理人员负责，因为其公司管理系统具有较强的特殊性，所以在设计系统时要具备安全性，确保系统中的数据是真实完整的，不会被非法人员轻易地修改或者删除等。
4.1.2 系统的设计原则
该系统必须具备的特点之一是可扩展性，它不仅相对容易导航，而且具有友好的界面，使其能在多个平台上运行平稳。

根据具体的业务需求，定制客户行动的具体类型和内容。 它还具有可视化功能，使用户能够操作和管理系统。系统还需要具有一定的健壮性。系统必须具备一定的扩展性。系统应该使后期管理人员能够进行良好地维护，同时在市场环境变化的时候可以开发新功能。系统还必须有良好的性能，具有确保用户在使用各种网络时延迟较低的特点。

图 5.1 系统登录模块
本模块的主要功能是提供控制器中的 LLER，LLER 负责处理 dispatcherservlet的分配要求。在业务处理级别进行处理之后，它将用户请求的数据装入模型，然后将该模型返回到对应的视图以供显示，写入一个方法。不必继承一个特定的类或者实现一个特定的接口[21]。
用户需要做的只是使用@控制器将类标记为控制器，然后通过@qrequestmapping的注释来定义 URL 请求与控制器方法的关联。这样，每个用户都能够方便地创建自己

所需的应用使客户端与服务器之间的信息能够得到有效填补。当用户输入请求时，系统就会根据这些请求自动执行相关操作。这样，控制器可以从外部访问，用户可以快速登录到页面。

6.3 测试方案
6.3.1 模块测试方案
（1） 功能性测试
在公司管理体系中，每个模块不仅是参与者，也是操作者，在多模块处理的过程中，对系统资源进行有效配置，确保公司管理系统的合作、安全和流畅[24]。在系统软件层面：促进功能测试对于管理系统的在线运行至关重要。员工应能够验证公司管理系统的准确性、效率和有用性，并可：因此，测试人员必须负责公司管理系统的独立功能，单独测试支架的功能模块。
（2） 兼容性测试
兼容性是指公司管理系统可以在不同的系统，例如在 a 平台开发，但用户可以在 b 平台进行使用。
（3） 易容性测试
简而言之，易容性很容易使用。对于系统的用户界面，它应该有一个字体大小模块。区块布局和颜色搭配应美观，以便用户在浏览时更加舒适经验丰富；此外，系统还应便于视觉操作，且系统应适当缩短快捷键，并相应地添加文本和照片作为有用的说明，以便用户了解系统的功能能够理解和操作相关的系统功能。
（4） 可靠性测试
可靠性测试主要是在运行过程中保持数据的一致性，这是主要的过程支持数据库技术的应用。
（5） 安全性测试
在系统中，安全性是需要强有力的保障，保证系统的数据在日常使用中不会被窃取，通过用户身份验证限制登录和用户访问，从而避免恶意的外部入侵，提高系统的稳定性。

结论

在现代信息技术的蓬勃发展之下，各种计算机技术迅速走进人们的日常生活，让人们不断助力于信息化社会的建设。由于人们现代商业组织的时代要求与未来发展需求，互联网技术逐渐融入现代公司企业，使现代公司企业在管理方面获得创新型的管理方式，其中公司管理系统作为具有代表性的现代软件，能够提高公司的服务水平并且让公司内部人员得到更加便利与高效的服务。本系统根据人们的实际需求，结合现代公司的实际运营情况，在整理之前的研究以及相关技术的基础上，使用 Java 语言和 MySQL 数据库，让这个公司管理系统实现。在 Java 代码的构建中，发现 Java 并不是很难，但是需要根据设计的需求进行逻辑结构的设计与实现，并对其他技术的融合开发却是一个有难度的条件，这不仅需要我的知识广度，还要求了我的知识深度。为了更加实现这个系统，我们将更加努力地去发挥我们的技术，让这个系统有更加完美的发展，使未来这个系统在市场被人所借鉴发展。在这个系统中，我们基本建立了一个具有多种功能的创慧公司内部管理系统。然而，由于我们的开发时间有限，对设计要求的理解不足，知识不足，系统功能并未充分展现。特别是系统的功能和数据库的逻辑结构仍有改进的余地。我需要继续学习解决系统中存在的问题。
